# Rocky Linux 9 + xrdp 環境で JIS キーボードを正しく認識させる手順 (ユーザー権限版)

## 概要
Rocky Linux 9 (GNOMEデスクトップ環境) に対して Windows からリモートデスクトップ (xrdp) 接続した際、手元のキーボードがJIS配列にもかかわらず、記号などがUS配列として入力されてしまう問題を解決するための手順です。

本手順は **管理者権限 (`root` / `sudo`) を持たない一般ユーザー** が、自身のログインセッション内のみ設定を上書きして解決する方法を記載しています。

## 環境要件
* **OS:** Rocky Linux 9 (GNOME デスクトップ環境)
* **接続方法:** Windows (11など) からのリモートデスクトップ接続 (xrdp)
* **権限:** 一般ユーザー

## 原因
WindowsのRDPクライアントは接続時にキーボードの「言語ID（日本語IMEの場合は `0411` など）」を送信しますが、Rocky Linux側の xrdp 共通設定（`/etc/xrdp`）に完全一致するマッピングファイルが存在しない場合、デフォルトの「US配列（`0409`）」としてフォールバック処理されてしまうためです。

---

## 解決手順

### 1. 自動起動用ディレクトリの作成
ログイン時に強制的にJIS配列へ書き換える処理を自動実行させるため、ユーザー固有の自動起動ディレクトリを作成します。ターミナルで以下のコマンドを実行します。

```bash
mkdir -p ~/.config/autostart

```

### 2. 自動起動設定ファイルの作成

作成したディレクトリの中に `.desktop` ファイルを作成します。

```bash
nano ~/.config/autostart/setxkbmap-jp.desktop

```

ファイルが開いたら、以下の内容を貼り付けて保存します。

```ini
[Desktop Entry]
Type=Application
Exec=setxkbmap -model jp106 -layout jp
Hidden=false
NoDisplay=false
X-GNOME-Autostart-enabled=true
Name=Set JIS Keyboard
Comment=Force JIS Keyboard layout for xrdp session

```

### 3. 日本語入力 (Mozc) の設定確認

GNOME環境の `ibus-mozc` がシステム設定を上書きしないよう確認します。

1. 画面右上の言語アイコンから **「日本語 (Mozc)」** を選択します。
2. ターミナルから以下のコマンドを実行し、Mozcの設定画面を開きます。

```bash
   /usr/libexec/mozc/mozc_tool --mode=config_dialog

```

3. 「一般」タブ内の **「キーボード配列」** を「システム標準」または「日本語」に設定し、「OK」をクリックします。

### 4. 再ログインと確認

設定を反映させるため、一度リモートデスクトップからログアウト（セッションを終了）し、再度接続します。
ターミナルやテキストエディタを開き、`@` や `:` などがキートップの印字通りに入力できれば設定完了です。

---

## ⚠️ 重要な注意事項

本設定はユーザーの「ログイン後」のデスクトップ画面にのみ適用されます。
xrdpの仕様上、**リモートデスクトップ接続直後の「ログイン画面（ユーザー名とパスワードを入力する画面）」はUS配列のまま**となります。

パスワードに記号（`@` や `*` など）を含んでいる場合は、ログイン画面に限りUS配列のキー配置を想定して入力する必要があります。（例：`@`を入力したい場合は `Shift + 2` を押す）
